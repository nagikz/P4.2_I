Práctica 4.2 - Github con Netbeans

Guion:
Crea una cuenta Github con un email no corporativo (Qué es GitHub).
Crea un repositorio con el nombre de proyecto igual al grupo que hayas elegido (por ejemplo: P4.3_A). EL repositorio lo debe crear el alumno que figure primero en la elección de cada grupo. Recuerda que el nombre del repositorio y el proyecto de Netbenas deben tener exactamente el mismo nombre (Crear un repositorio en GitHub).
Debes dar acceso al repositorio al resto de los miembros del grupo.
Desde Netbeans haz alguna modificación (añade ficheros, etc y haz un commit con una versión nueva)
Sube el repositorio desde Netbeans a Github (Subir repositorio en GitHub con NetBeans)
Ojo que con el cambio de política de seguridad de Github necesitará crear un Token para poder subir la versiones a Github (Solución problema subir cambios a GitHub con Netbeans - Token de Acceso). Elige la opción del token fine-grained. 
Compartir el proyecto con el resto de alumnos del grupo. El resto de alumos deben clonar el proyecto en Netbeans  y hacer cambios y realizar un commit, poner en la descripción: Cambio realizado por el alumno "Nombre y apellidos"),  Luego deben subir los cambios al repositorio y ver que ambos pueden actualizar edesde Netbeans el proyecto en github.
Crea un documento con las capturas de pantalla de las configuraciones y la resolución de los diferentes apartados. Para aprobar la práctica cada alumno debe mostrar en clase el funcionamiento del control de cambios en proyecto
